**Логика работы**
Каждый день, в 23:45 бот присылает отчет по показателям за текущую смену в группу Telegram, в тему города.
Кажду неделю, в воскресенье, в 23:50 бот присылает отчет по показателям за текущую неделю в группу Telegram, в тему города.
Каждый месяц, в последний день месяца, в 23:55 бот присылает отчет по показателям за текущий месяц в группу Telegram, в тему города.

Так же, есть возможность запросить отчет вручную, отправив команды в личные сообщения боту. По командам, можно получить отчет за текущую смену, за определенную дату и за определенный период.

**Формат отчетов**
Отчет в себе содержит следующие показатели:
Количество заказов;
LFL количество заказов (сравнение с текущим периодом. То есть если день, то аналогичный день в прошлой недел, если неделя, то сравнение с предыдущей неделей, если месяц, то сравнение с предыдущим месяцем).
Средний чек;
Выручка в разрезе каналов (Зал, Самовынос, Яндекс.Еда, Доставка курьером);
Общая выручка;
LFL выручка (сравнение с текущим периодом. То есть если день, то аналогичный день в прошлой недел, если неделя, то сравнение с предыдущей неделей, если месяц, то сравнение с предыдущим месяцем).
Дисконт в процентах (разница между выручкой со скидкой и без скидки);
Дисконт в сумме (разница между выручкой со скидкой и без скидки);
Стоп листы (стопы которые были в указанном периоде. То есть если это недельный отчет, то все стопы, которые были в течение этой недели, либо если указали какой то определенный период, то стопы за этот указанный период).

**Поля для получения показателей из OLAP**
Количество заказов - UniqOrderId
Средний чек - DishDiscountSumInt.average
Выручка в разрезе каналов - OrderType
Общая выручка - DishDiscountSumInt
Дисконт в процентах - DiscountPercent
Дисконт в сумме - DiscountSum

Выше прописаны поля, которые отвечают за тот или иной показатель в OLAP отчете. Нужно их использовать, так же воспользоваться формулами для расчета того или иного показателя, чтобы все было верно. Все получаемые отчеты, должны быть одинаковы, за исключением периода, по которому получаем отчет.

**Команды для получения отчетов вручную**
Отчеты можно получить с помощью следующих команд, они будут отправлять в ЛС откуда запрашиваем эти отчеты, они не будут дублироваться в группу:
/revenue {id} - получить отчет за текущую смену
/revenue {id} {date} - получить отчет за определенную дату. Формат даты: YYYY-MM-DD
/revnue {id} {dateFrom} {dateTo} - получить отчет за определенный период. Формат даты: YYYY-MM-DD

Отчеты можно запросить с помощью тестовых команд, они будут отправляться в группу, то есть мы просто автматические отчеты запрашиваем вручную:
/test_daily - запросить тестовый ежедневный отчет
/test_weekly - запросить тестовый еженедельный отчет
/test_monthly - запросить тестовый ежемесячный отчет
{id} - это идентификатор города, за который нужно получить отчет.

Бот должен работать стабильно, никаких импровизаций при запросе и отправке отчета, всегда цифры получать с API OLAP, данные всегда свежие должны быть и не отправлять много запросов по API чтобы оптимизировать все.
