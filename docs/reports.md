FoodMiniApp · 2025

# **Введение**

Данный документ является технической спецификацией аналитических отчётов для дашборда FoodMiniApp, построенного на данных iiko. Для каждого отчёта описаны: цель, источники данных, OLAP-поля, агрегаты, фильтры и рекомендуемая визуализация.

## **Условные обозначения источников**

| **Transport / Biz**   | iikoTransport API / iikoBizAPI - облачные данные доставки. Для новых проектов использовать iikoTransport. |
| --------------------- | --------------------------------------------------------------------------------------------------------- |
| **OLAP SALES**        | iikoServer API /resto/api/v2/reports/olap с reportType=SALES. Продажи, блюда, чеки, скидки.               |
| **OLAP TRANSACTIONS** | iikoServer API /resto/api/v2/reports/olap с reportType=TRANSACTIONS. Зарплаты, премии, выплаты.           |
| **Employees API**     | iikoServer /resto/api/employees - сотрудники, роли, явки (/attendance), расписание (/schedule).           |

⚠ Важно: точные имена OLAP-полей обязательно проверять через /resto/api/v2/reports/olap/columns перед разработкой. Названия полей могут отличаться в зависимости от версии iiko.

## **1.2 Контроль времени (приготовление / полка / в пути)**

| **Цель**            | Мониторинг соблюдения нормативов на каждом этапе доставки: кухня → сборка → курьер. |
| ------------------- | ----------------------------------------------------------------------------------- |

### **Поля и параметры**

| **Поле**            | **Тип/Источник** | **Назначение**                          |
| ------------------- | ---------------- | --------------------------------------- |
| orderId             | Transport        | Идентификатор заказа                    |
| createdAt           | Transport        | Время создания заказа                   |
| confirmedAt         | Transport        | Время подтверждения оператором          |
| Delivery.PackedTime | Server (iiko 9+) | Метка сборки заказа (с версии iiko 9.0) |

| deliveredAt         | Transport        | Время фактической доставки клиенту      |

| attendanceStart / End    | Employees /attendance       | Явки для расчёта часов                                 |
| scheduleStart / End      | Employees /schedule         | Плановые смены                                         |

**Примечания:** Типы транзакций для ФОТ: • BONUS - разовые премии • TARIFF_HOUR - почасовая оплата • INCENTIVE_PAYMENT - мотивационные выплаты Доля ФОТ = Sum(Sum.Incoming) / fullSum × 100%. Это один из самых сложных отчётов - требует корректного проектирования ключей связки departmentId / employeeId.

# **Приложение: Ключевые фильтры и правила**

- DeletedWithWriteoff = DELETED_WITHOUT_WRITEOFF - исключать из всех SALES-отчётов. Иначе удалённые без списания позиции искажают счётчики.
- DishType IN (DISH, MODIFIER) - стандартный фильтр для продуктовых отчётов. SERVICE-позиции (доставка, упаковка) считать отдельно.
- Быстрые кассовые заказы (< 3 мин) - исключать из SLA-отчётов по доставке.
- Заказы без оплаты - исключать из финансовых и SLA-отчётов.
- Поле fullSum (выручка) - обязательно проверять через /olap/columns, название может быть иным в вашей версии iiko.
- Ключи связки гибридных отчётов: Department.Id / Department.Code (OLAP) ↔ departmentId (Transport/Employees); Counteragent.Id (TRANSACTIONS) ↔ employeeId (Employees).
- OLAP TRANSACTIONS: нужные типы транзакций для ФОТ - BONUS, TARIFF_HOUR, INCENTIVE_PAYMENT.
- iikoBizAPI устарел и не подключается для новых клиентов - использовать iikoTransport API.
