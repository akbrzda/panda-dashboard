# Итоговые правила расчета метрик

Дата актуализации: 2026-04-19

## 1. Базовая выборка заказов

- Для пользовательских отчетов по продажам, выручке, клиентам и KPI по умолчанию используются только завершенные заказы.
- Отмененные/сторнированные заказы исключаются из пользовательского контура.
- Единые правила применяются на backend и переиспользуются в отчетах Dashboard, Revenue, Clients, SLA и Delivery Delays.

## 2. Late/SLA и доставка

- Late-логика и доля опозданий рассчитываются по единому правилу сравнения `actual` vs `promised`.
- Для SLA используется all-orders выборка с общим правилом обработки missing promised/actual timestamps.
- Полнота данных (сколько заказов исключено из SLA/late) отражается в отчетах доставки.

## 3. Дисконт и LFL

- Процент и сумма дисконта используют единый знаменатель и единый набор исходных полей.
- LFL для выручки и заказов применяется консистентно в Revenue и Dashboard.
- Для среднего чека и дисконта действует та же LFL-база сравнения, что и для выручки.

## 4. Клиентская аналитика

- UI-фильтры в отчете Clients ограничены бизнес-базой: организация + период + сегментация.
- Технические фильтры (terminal group, статусы, enrichment mode, profile limit) выведены из пользовательского контура.
- Принудительное обновление без кэша выполняется из стандартного действия в шапке страницы.

## 5. Планы и прогноз

- Месячный план выручки задается одной суммой и раскладывается по дням автоматически.
- Дневные веса строятся по исторической загрузке за авто-окно анализа (`PLANS_MONTHLY_ANALYSIS_DAYS`, по умолчанию 20 дней, допустимо 7-90).
- Сумма дневной раскладки всегда равна общему месячному плану (с корректировкой округления на последний день).
- Прогноз загрузки производства принимает только будущую `forecastDate`.
- Исторический период для прогноза выбирается автоматически по окну (`REPORTS_FORECAST_ANALYSIS_DAYS`/`analysisWindowDays`, 7-90 дней).

## 6. Имена подразделений

- Значения `Unknown/unknown/Неизвестно/null/undefined` не выводятся в UI как имя подразделения.
- Если валидное имя отсутствует, используется человекочитаемый fallback: `Подразделение <id>` или `Подразделение без названия`.
