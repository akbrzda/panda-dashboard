# Финальный регресс-чеклист

Дата: 2026-04-19

## 1. Сверка ключевых метрик между отчетами

- [ ] Выбрать 2 контрольные даты и 1 подразделение.
- [ ] Сверить `totalRevenue`, `orders`, `avgCheck`, `discountSum` между Dashboard и Revenue.
- [ ] Сверить late-метрики между Delivery Delays, SLA и Courier KPI.
- [ ] Зафиксировать расхождения >1% (или >1 заказ для малых выборок) с причиной.

## 2. Контроль статусов заказов

- [ ] Убедиться, что по умолчанию отчеты используют завершенные заказы.
- [ ] Проверить, что отмененные/сторнированные заказы не влияют на пользовательские KPI.
- [ ] Проверить одинаковое поведение completed-only фильтра в Revenue, Dashboard, Clients.

## 3. LFL и дисконт

- [ ] Проверить LFL-показатели: выручка, заказы, средний чек, дисконт.
- [ ] Убедиться, что формулы и единицы измерения консистентны между Dashboard и Revenue.

## 4. Интерфейс без DEV-тегов

- [ ] На пользовательских страницах нет меток `Planned/Partial/Beta/DEV`.
- [ ] Нет технических блоков `Updated at`, `Tier`, `Source`, `Coverage`, `Last reviewed`.
- [ ] Кнопка обновления данных отображается как стандартное действие в шапке отчета.

## 5. Мобильная и desktop верстка

- [ ] Проверить страницы: Clients, Plans, Production Forecast на ширинах `360px`, `768px`, `1280px`.
- [ ] Убедиться, что после удаления/переноса блоков не появились пустые зоны и горизонтальный скролл.
- [ ] Проверить таблицы: читаемость, переносы, доступность основных колонок.

## 6. Планы и прогноз

- [ ] В `Plans` проверить сценарий месячного плана одной суммой.
- [ ] Проверить дневную раскладку: дата, план на день, доля (%), сумма по дням = общий план.
- [ ] Проверить fallback названий подразделений: отсутствуют `Unknown/unknown`.
- [ ] В `Production Forecast` доступен ввод только будущей даты прогноза.
- [ ] Проверить, что метаданные прогноза содержат авто-выбранное окно анализа.
