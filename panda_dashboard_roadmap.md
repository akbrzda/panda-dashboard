## Фаза 1 — Критические исправления

> Безопасность, целостность данных, баги влияющие на доверие к продукту.  
> **Оценка: ~15 ч**

### Безопасность и доступ

- [ ] Добавить базовую аутентификацию (API-key middleware на все роуты) — `4 ч` · **Критично** · Логика
- [ ] Сделать `IIKO_WEBHOOK_TOKEN` обязательным, отклонять запросы без токена — `1 ч` · **Критично** · Логика
- [ ] Ограничить CORS с `*` на конкретный origin (env-переменная) — `30 мин` · Средний · Логика

### Целостность данных — plans.json

- [ ] Заменить `fs.writeFile` на `write-file-atomic` (атомарные записи) — `2 ч` · **Критично** · Логика
- [ ] Добавить mutex (`async-mutex`) для сериализации операций с файлом — `1 ч` · **Критично** · Логика
- [ ] Реализовать rollback при ошибке optimistic delete в `PlansStore` — `1 ч` · Средний · Логика

### Критические баги UI

- [ ] Dashboard: показывать предупреждение при нулевых данных из-за ошибок API (`hasErrors: true`, список недоступных org) — `2 ч` · **Критично** · Логика
- [ ] Защита LFL от деления на ноль (`previous === 0` → `return null`, LFLBadge не отображается) — `30 мин` · **Критично** · Логика
- [ ] Исправить мигание контента при `reset()` стора (пустое состояние вместо skeleton) — `1 ч` · Средний · UX

## Фаза 2 — Quick Wins UX и стоп-лист

> Быстрые победы с высоким эффектом и малыми усилиями.  
> **Оценка: ~10 ч**

### Quick wins UX (до 1 ч каждая)

- [ ] Русифицировать бейджи навигации: `Ready→Готово`, `Beta→Бета`, `Partial→Частично`, `Planned→Планируется` — `30 мин` · **Критично** · UX
- [ ] Заблокировать переход на planned-страницы (`pointer-events-none` + страница-заглушка с пояснением) — `1 ч` · **Критично** · UX
- [ ] Добавить toast «Данные обновлены» после нажатия Refresh (через Sonner) — `30 мин` · Низкий · UX
- [ ] Добавить tooltip с объяснением LFL рядом с фильтром — `1 ч` · Средний · UX
- [ ] Унифицировать заголовки `TopDishesView` и `PlansView` под стандарт `ReportPageHeader` — `1 ч` · Средний · UX

### Стоп-лист: быстрые исправления

- [ ] Алертинг в Telegram: scheduler → `/api/stop-lists` → фильтр по `inStopHours > 2 AND PLR > 0` → уведомление — `3 ч` · **Критично** · Стоп-лист
- [ ] Защита от concurrent refresh (promise deduplication в `StopListService`) — `30 мин` · Средний · Стоп-лист
- [ ] Валидация `timezone` через `Intl.supportedValuesOf` и UUID-формата `organizationId` — `30 мин` · Средний · Стоп-лист
- [ ] Warning при `endedAt < startedAt` в `getDuration()` — `15 мин` · Низкий · Стоп-лист
